How it works 03 steps

Three commands. One dashboard.

No account. No server. Works inside your existing CI pipeline.

01 · Connect

Point at your tool

Run npx certainty-units init to create a config file. Add your API key and team ID — two lines, done.

02 · Sync

Fetch and score

certainty-units pulls every issue, maps your status fields, and computes a 0–100 certainty score from six signals: validation, progress, evidence, criteria, sizing, and discussion.

03 · Ship

Open the report

A self-contained cu-report.html lands in your folder. Open locally, commit to the repo, or auto-publish to GitHub Pages on a schedule.

Scoring model 6 signals · 0–100

What goes into a Certainty Score

Six signals, each weighted. The score tells you not just what's done — but what's understood.

40 pts

Validation status

validated → 40 · assumed → 10 · needs_clarification → 5

20 pts

Workflow progress

done → 20 · review → 15 · in_progress → 10

10 pts

Acceptance criteria

Item has a clear definition of done — someone thought through what "finished" means.

10 pts

CU tier set

basic · intermediate · advanced — signals deliberate sizing by the team.

5 pts

Evidence / description

Non-empty description — the team wrote something down about the item.

+15 pts

Discussion activity

Comment count × 5, capped at 15. Discussed items carry less hidden risk.

Configuration

Field mapping is yours to control

Your team calls it "Shipped" instead of "done"? Map it. One YAML file, no code changes.

project: My Project
source:  linear   # linear | jira | notion

linear:
  apiKey: ${LINEAR_API_KEY}   # env var, never committed
  teamId: your-team-id

  fieldMap:               # optional — override defaults
    cu_tier:
      field: estimate      # map story points to CU tiers
      map:
        1: basic
        3: intermediate
        8: advanced

output:
  html:     cu-report.html
  markdown: cu-report.md
