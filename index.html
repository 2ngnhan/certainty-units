How it works

Three commands. One dashboard.

No account. No server. Works inside your existing CI pipeline.

01 · Connect

Point at your tool

Run npx certainty-units init to generate a config file. Add your API key and team ID.

02 · Sync

Fetch and score

Pulls every issue, maps your status fields, and computes a 0–100 certainty score from six signals.

03 · Ship

Open the report

A self-contained cu-report.html in your folder. Open locally or publish to GitHub Pages.

Scoring model

Six signals. One score.

The certainty score tells you not just what's done — but what's understood.

40 pts

Validation status

validated → 40 · assumed → 10

20 pts

Workflow progress

done → 20 · in_progress → 10

10 pts

Acceptance criteria

Item has a clear definition of done.

10 pts

CU tier set

basic · intermediate · advanced

5 pts

Evidence

Non-empty description field.

+15 pts

Discussion

Comment count × 5, capped at 15.

Configuration

Field mapping is yours to control

Your team calls it "Shipped" instead of "done"? Map it. One YAML file.

project: My Project
source:  linear

linear:
  apiKey: ${LINEAR_API_KEY}
  teamId: your-team-id
  fieldMap:
    cu_tier:
      field: estimate
      map: { 1: basic, 3: intermediate, 8: advanced }

output:
  html: cu-report.html
