How it works
Three commands. One dashboard.
No account. No server. Your API key stays local.
01  CONNECT

Point at your tool

Run npx certainty-units init to generate a config file. Add your API key and team ID — that's it.

02  SYNC

Fetch and score

certainty-units pulls every issue, maps your status fields, and computes a 0–100 certainty score based on validation, workflow progress, evidence, and discussion activity.

03  SHIP

Open the report

A self-contained cu-report.html lands in your project folder. Open it locally, commit it, or auto-publish to GitHub Pages weekly.

Scoring model
What goes into a Certainty Score
Six signals, each weighted. The score tells you not just what's done — but what's understood.
40 pts

Validation status

validated → 40 · assumed → 10 · needs_clarification → 5

20 pts

Workflow progress

done → 20 · review → 15 · in_progress → 10

10 pts

Acceptance criteria

Item has a definition of done — someone thought through what "finished" means.

10 pts

CU tier set

basic / intermediate / advanced — signals deliberate sizing.

5 pts

Evidence / description

Non-empty description — the team wrote something down about it.

+15 pts

Discussion activity

Comment count × 5, capped at 15 — discussed items carry less hidden risk.

Configuration
Field mapping is yours to control
Your team calls it "Shipped" instead of "done"? Map it. One YAML file, zero code.
project: My Project
source:  linear   # linear | jira | notion

linear:
  apiKey: ${LINEAR_API_KEY}   # env var expanded at runtime
  teamId: your-team-id

  fieldMap:                   # optional overrides
    cu_tier:
      field: estimate
      map:
        1: basic
        3: intermediate
        8: advanced

output:
  html:     cu-report.html
  markdown: cu-report.md
